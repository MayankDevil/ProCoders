Git

Git is a distributed version control system is a tool that records and manages changes in a project during software development.

  • Keep a history of project changes.
  • Work on different features using branches.
  • Easy to Collaborate with other developers.
  • Restore previous versions if something goes wrong.
  • Manage code efficiently using repositories.

Git Terms

  • Repository (repos) : Your project folder tracked by Git
  • Local Repository : Project stored on your computer
  • Remote Repository : Project stored on a server like GitHub
  • origin : Default name of the main remote repository
  • Branch : A version/line of development inside a repository

Commands

  1. Initalize (.git)

    which local repos you make (__.git__)

    git init
  2. Add File & Folder

    new or modify file add in stage

    git add folder/file

    every modification add in stage

    git add .
  3. Commit

    committed this modification is stay with message

    git commit -m "message"

    add in stage or committed this modification is stay with message

    git commit -a "add or commit both"
  4. Status

    present position of current repos

    git status
  5. Remove File

    remove files and modify stage

    rm folder/file
    • add all file to stage
    • commit to save
  6. Delete File & Folder

    delete file and stage automatically

    git rm file

    delete folder and stage automatically

    git rm -r folder
    • commit to save
  7. Erase (.git)

    destory all git logs in local

    rm -rf .git

Clean

Show untracked files

git clean -n

delete untracked files only

git clean -f

delete untracked files and folders

git clean -fd

Git Branch

View

show all branches of local

git brach -a

show all branches of remote

git brach -r

Switch

switch control to other branch

git switch branch_name

simple way to checkout

git checkout branch_name

Create

create new branch and switch into it

git checkout -b new_branch_name

if brach exist then switch else before switch create branch

git switch -c new_branch_name

Delete

delete branch if merged

git branch -d new_branch_name

delete branch by force

git branch -D new_branch_name

delete branch of remote origin

git push origin_name --delete branch_name

Git Undo

Reset

git reset HEAD~1
  • committed removed
  • Files remain in working folder

Soft Reset

use when you committed by mistake

git reset --soft HEAD~1
  • commited removed
  • changes stay staged

Hard Reset

use when you erase to last safe commit

git reset --hard HEAD~1

also hard reset by specific id

git reset --hard commit_id

Git Log

view full commit history

git log
  • commit id
  • author name
  • commit date
  • commit message

compact history view in one line

git log --oneline

view log by author

git log --author="username"

view log by commit message

git log --grep="keyword"

view log after specific date

git log --since="YYYY-MM-DD"

view log before specific date

git log --until="YYY-MM-DD"

detail log history

proper present commit log history and stop more show by (PRESS - Q)

git show commit_id
