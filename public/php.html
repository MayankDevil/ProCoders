Control Statement

In programming, sometimes we need to execute different blocks of code depending on a specific condition. Control Statements help developers control the flow of program execution. They decide which code should run and which code should skip based on conditions.

If else Statement

The if statement executes a block of code only when the given condition is true. If the condition is false, the else block will execute.

if (expression) { #true }
OR
if (expression) { #true } else { #false }

Control Flow

  1. The program first evaluates the if condition.
  2. If the condition result is true, the if block code will execute.
  3. After executing the if block, the program continues with the next statement.
  4. If the condition result is false, the if block will be skipped.
  5. In an if else statement, the else block will execute when the condition is false.
  6. After completion, program execution continues with the remaining code.
<?php

    $isLogin = false;

    if ($isLogin) {
        echo "Welcome UserName";
    } else {
        echo "Welcome Guest";
    }

    $isLogin = true;

    if ($isLogin) {
        echo "add logout";
    }
?>
Welcome Guest
add logout

Switch Statement

The switch statement is used when we need to compare one value with multiple possible values. It provides a cleaner way to write multiple else if conditions.

swtich (experssion) { case value1 : //code case value2 : // code default : //code }

Control Flow

  1. The program first evaluates the switch expression.
  2. The value is compared with each case value one by one.
  3. If a matching case is found, that case block will execute.
  4. After executing the case block, the break statement stops the switch.
  5. If no case matches, the default block will execute.
  6. After completion, program execution continues with the next statement.
<?php

    $day = "Monday";

    switch ($day) {

        case "Monday": echo "Today is Monday";
            break;

        case "Tuesday": echo "Today is Tuesday";
            break;

        case "Wednesday": echo "Today is Wednesday";
            break;

        default: echo "Invalid Day";
    }
?>
Today is Monday

For Loop

The for loop is used when we know how many times we want to execute a block of code. It repeats the code until the given condition becomes false.

for (initialization; condition; increment) { //code }

Control Flow

  1. The program initializes the loop variable.
  2. The condition is checked before every iteration.
  3. If the condition is true, the loop block executes.
  4. After execution, the increment/decrement operation runs.
  5. The loop continues until the condition becomes false.
  6. After completion, program execution moves to the next statement.
<?php

    for ($i = 1; $i <= 5; $i++) {

        echo "Number : " . $i . "<br>";
    }

?>

While Loop

The while loop executes a block of code as long as the given condition remains true.

while (condition) { //code }

Control Flow

  1. The program checks the while condition.
  2. If the condition is true, the loop block executes.
  3. After execution, the condition is checked again.
  4. The loop continues until the condition becomes false.
  5. If the condition is false, the loop stops.
<?php

    $i = 1;

    while ($i <= 5) {

        echo "Number : " . $i . "<br>";
        $i++;
    }

?>

Do While Loop

The do while loop executes the code block first and checks the condition after execution. It always runs at least one time.

do { //code } while (condition);

Control Flow

  1. The program executes the loop block first.
  2. After execution, the condition is checked.
  3. If the condition is true, the loop runs again.
  4. If the condition is false, the loop stops.
  5. The code executes at least one time even if the condition is false.
<?php

    $i = 1;

    do {
        echo "Number : " . $i . "<br>";

        $i++;

    } while ($i <= 5);

Object Oriented Programming (OOP)

Object Oriented Programming is a programming approach that organizes code using objects and classes. It helps developers write reusable, secure, and maintainable code.

Class

A class is a blueprint or template used to create objects. It defines properties (variables) and methods (functions) that an object can use.

class ClassName { // properties and methods }

Object

An object is an instance of a class that created using the new keyword.

$object = new ClassName()
  • The program creates a class as a blueprint.
  • The class contains properties and methods.
  • The object can access class properties and methods.
  • Multiple objects can be created from one class.
<?php

class Client
{
    // Property
    public $name;

    // Method
    public function message()
    {
        echo "Welcome " . $this->name;
    }
}
// Object

$c1 = new Client();
$c2 = new Client();

$c1->name = "Mayank"; // Access Property
$c1->message(); // Access Method

$c2->name = "Kali"; // Access Property
$c2->message(); // Access Method

Constructor and Destructor

A constructor is a special method that automatically runs when an object is created. It is mainly used to initialize object properties.

A destructor is a special method that automatically runs when an object is destroyed or the script finishes. It is used to perform cleanup tasks.

__construct() { //object initialize code }
OR
__destruct() { //after end code }

Constructor Flow

  1. The program creates a class.
  2. An object is created using the new keyword.
  3. The __construct() method automatically executes.
  4. Initial values are assigned to object properties.
  5. The object is ready to use.

Destructor Flow

  1. The object finishes its work.
  2. The script ends or the object is removed.
  3. The __destruct() method automatically executes.
  4. Cleanup operations are performed.

Inheritance

Inheritance is an OOP feature that allows one class to use the properties and methods of another class. It helps to reuse existing code and reduce duplication.

The class which provides properties and methods is called the Parent Class, and the class which receives them is called the Child Class.

class Child extends Parent { // code }

Inheritance Flow

  1. The program creates a parent class with common properties and methods.
  2. A child class uses the extends keyword.
  3. The child class inherits parent class members.
  4. The child class can use inherited methods and properties.
  5. The child class can add its own properties and methods.
<?php

class Person
{
    public $id;
    public $name;

    public function setId($id) {

        $this->id = $id;
    }
    public function setName($name) {

        $this->name = $name;
    }
}

class Employee extends Person
{
    public salary;
    
    public function status () {

        echo "Employee " . $this->id . " name is " .$this->name . " and salary is " . $this->salary . "<br>";
    }
}
$emp = new Employee();

$emp->setId(3);
$emp->setName("Kalid");

$emp->salary = 30000;

$emp->status();

?>

Employee 3 name is Kalid and salary is 30000

Method Overriding

  • Method overriding happens when a child class creates a method with the same name as the parent class method.
  • The method name in both parent and child class must be the same.
  • The child class method replaces the parent class method when called using the child object.
  • Method overriding is used to change or customize parent class behavior.
  • The access modifier of the child method cannot be more restrictive than the parent method.
accessmodifer function methodName (args) { //Override super class code }

        echo "Admin User";

$obj->role();

Cookies

A cookie is a small piece of data stored in the user's browser. Cookies are used to remember user information between requests, such as login status, preferences, and other settings.

setcookie(name, value, expire, path);

Cookie Flow

  1. The server creates a cookie using the setcookie() function.
  2. The cookie is stored inside the user's browser.
  3. The browser sends the cookie back with every request to the same website.
  4. PHP reads the cookie using the $_COOKIE super global array.
  5. The server uses cookie data to remember user information.
  6. A cookie can be removed by setting an expired time.
<?php

    setcookie(
        "theme",
        "dark",
        time() + 3600,
        "/"
    );
    echo "Cookie Created";

?>
Cookie Created

SERVER Variable

The $_SERVER is a PHP super global variable that contains information about the server, request, and execution environment. It provides details like current file path, request method, browser information, and server details.

$_SERVER
OR
$_SERVER['keyName'];

Server Variable Flow

  1. The user sends a request to the web server.
  2. The server processes the request.
  3. PHP creates the $_SERVER array with request information.
  4. The program accesses required server data using keys.
  5. PHP returns the response to the user.
  • $_SERVER["PHP_SELF"] - Returns current executing file name.
  • $_SERVER["SERVER_ADDR"] - Returns the IP address of the server.
  • $_SERVER["SERVER_NAME"] - Returns the server host name.
  • $_SERVER["SERVER_SOFTWARE"] - Returns server software information.
  • $_SERVER["SERVER_PROTOCOL"] - Returns the name and version of the protocol.
  • $_SERVER["REQUEST_METHOD"] - Returns request method (GET, POST, PUT, DELETE).
  • $_SERVER["REQUEST_TIME"] - Returns timestamp when request started.
  • $_SERVER["REQUEST_TIME_FLOAT"] - Returns request start time with microseconds.
  • $_SERVER["QUERY_STRING"] - Returns query string from URL.
  • $_SERVER["DOCUMENT_ROOT"] - Returns root directory of current website.
  • $_SERVER["SCRIPT_FILENAME"] - Returns full path of current script.
  • $_SERVER["SCRIPT_NAME"] - Returns current script path.
  • $_SERVER["SCRIPT_URI"] - Returns full URL of current script.
  • $_SERVER["HTTP_HOST"] - Returns current host name.
  • $_SERVER["HTTP_USER_AGENT"] - Returns browser and device information.
  • $_SERVER["HTTP_ACCEPT"] - Returns accepted content types.
  • $_SERVER["HTTP_ACCEPT_LANGUAGE"] - Returns user's language preference.
  • $_SERVER["HTTP_REFERER"] - Returns previous page URL.
  • $_SERVER["REMOTE_ADDR"] - Returns user's IP address.
  • $_SERVER["REMOTE_HOST"] - Returns user's host name.
  • $_SERVER["REMOTE_PORT"] - Returns user's port number.
  • $_SERVER["HTTPS"] - Checks whether HTTPS connection is enabled.
  • $_SERVER["GATEWAY_INTERFACE"] - Returns CGI version used by server.
  • $_SERVER["SERVER_PORT"] - Returns server port number.
  • $_SERVER["SERVER_ADMIN"] - Returns server administrator email.
  • $_SERVER["PATH_INFO"] - Returns extra path information.

  2. The browser sends file data to the server.

  5. The file can be saved, renamed, or validated.
<?php

if(isset($_POST['submit'])) {

        pathinfo($fileName, PATHINFO_EXTENSION)
    );

        // Check upload error

    } else if(!in_array($extension, [ "jpg", "jpeg", "png", "gif"])) {

        // Check size (2MB)

        // New unique file name

Database Connection Class

A database connection class is used to create a reusable connection between PHP and MySQL database. It avoids writing database connection code again and again.

  1. Database Connection

    The mysqli class creates a connection between PHP application and MySQL database using database credentials.

    new mysqli(hostname, username, password, database)
  2. Connection Error

    The connect_error property checks whether the database connection failed and returns the error message.

    connection_object->connect_error
  3. Database Operation

    After successful connection, SQL queries can be executed to insert, update, delete, or fetch database records.

    connection_object->query($sql)
  4. Connection Close

    The close() method closes the database connection and releases the connection resources.

    connection_object->close() 
<?php

/* Connection Establised */

$db = new mysqli(
    hostname,
    username,
    password,
    database
);

/* Connection Status Verified */

if ($db->connect_error) {
    
    die("connection Unestablished");

} else {

    // All database query execute or data fetch between this

}

/* Connection Closed */

$db->close();

$db = null;

?>
  1. Prepare Query

    The prepare() method creates a prepared statement from an SQL query. It helps to execute queries safely and prevents SQL injection.

    connection_object->prepare(query)
  2. Bind Values

    The bind_param() method attaches values to placeholders (?) in the prepared query.

    statement_object->bind_param(type, values, ...)
  3. Execute Statement

    The execute() method runs the prepared SQL statement with the provided values.

    statement_object->execute()
  4. Statement Error

    The error property stores the error message if prepare or execute operation fails.

    statement_object->error
  5. Get Result

    The get_result() method returns the result object after executing a SELECT prepared statement.

    result_object = statement_object->get_result()
  6. Close Statement

    The close() method releases the prepared statement and frees resources.

    statement_object->close()

Header Function

The header() function is a built-in PHP function used to send raw HTTP headers from the server to the browser. It gives instructions to the browser about how to handle the response.

header("header-name: value");

What is HTTP Header?

  • HTTP headers are information sent between server and browser with every request and response.
  • Headers contain details like content type, redirection, cache rules, authentication, and file information.
  • PHP uses header() to modify these response headers.

Header Rules

  • The header() function must be used before any output.
  • HTML code, echo, print, or whitespace before header() can cause "headers already sent" error.
  • Multiple headers can be sent in one PHP file.
  • After sending a redirect header, use exit or die to stop further execution. 
<?php

// redirect page

header("Location: dashboard.php");

exit;

// Browser understands that response is HTML.

header("Content-Type: text/html");

// JSON API Response

Include and Require

The include and require statements are used to insert the content of one PHP file into another PHP file. They help to reuse code and organize large applications.

include "file.php";
OR
require "file.php";

Include and Require Rules

  • include gives a warning if the file is not found, but script execution continues.
  • require gives a fatal error if the file is not found, and script execution stops.
  • Both statements can be used for reusable files like database connection, functions, and classes.
  • The file path can be relative or absolute.
